Notes and Stuff

  • Important to ensure all required forms are filled out
  • Client-side form validation helps ensure submitted data matches requirements
  • It's not the end all be all of validation
  • Validation done in the browser is client-side, while server-side is obviously validated on the server
    • Two types of client-side validation:
    • Built-in form validation - uses HTML form validation features
    • JavaScript validation - completely customizable validation coded in JavaScript

    • Built-in form validation example:



    Passwords must be at least 8 characters and contain at least 1 number, 1 capital letter and 1 special character

    Validating using JavaScript

  • Most browsers support the Constraint Validation API, which has a bunch of nifty prebuilt methods and properties to help you validate
